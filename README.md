# Smart City Surveillance System

This project simulates a Smart City Surveillance System implemented using Event-Driven Architecture (EDA) and the Publish–Subscribe pattern using an in-process EventBus.

The system processes real-time events generated by different types of surveillance cameras and distributes them to multiple independent services such as police departments, navigation apps, and a city dashboard.

---

## Architecture

The system is based on the Publish–Subscribe model:
- Cameras act as **Publishers**
- Services act as **Subscribers**
- EventBus acts as the **Event Broker**

Publishers do not communicate directly with subscribers. All communication is done through the EventBus, which ensures loose coupling and allows new services to be added without modifying existing components.

---

## Event Types

The system uses three types of events:

- **SpeedingVehicleEvent** – generated when a speed camera detects a vehicle exceeding the speed limit.
- **TrafficCongestionEvent** – generated when a traffic camera detects congestion.
- **GarbageThrowingEvent** – generated when a public surveillance camera detects illegal garbage throwing.

Each event contains relevant data such as camera ID, neighborhood, street/location, and specific event details.

---

## Components

### Publishers
- SpeedCamera
- TrafficCamera
- PublicSurveillanceCamera

### Subscribers
- TrafficPolice – issues fines for speeding
- LocalPolice – issues fines for garbage throwing
- NavigationApp – receives traffic congestion updates
- CityDashboard – displays statistics and most dangerous neighborhoods

### Event Broker
- EventBus (Greenrobot EventBus)

---

## Simulation

The simulation is executed from `MainSmartCity.java`, where:
- Subscribers are registered to the EventBus
- Multiple cameras (publishers) generate events
- Events are delivered to all interested subscribers
